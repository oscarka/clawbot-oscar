# OpenClaw 自动化解决方案指南

## 概述

本文档提供完全自动化的解决方案，无需手动干预即可处理卡顿问题。

## 自动化方案

### 方案一：自动监控和修复

创建一个监控脚本，定期检查系统状态，发现卡顿时自动处理。

**工作原理：**
- 每 5 分钟自动检查一次 Gateway 状态
- 检测到超时任务时，自动发送停止命令
- 检测到连续超时时，自动重置会话
- Gateway 无响应时，自动重启服务

**优势：**
- 完全自动化，无需人工干预
- 可以自定义检测阈值
- 记录所有操作到日志文件

**设置方法：**
1. 脚本已创建在：openclaw_auto_monitor.sh
2. 添加到系统定时任务（crontab）
3. 设置每 5 分钟执行一次

**监控内容：**
- Gateway 进程是否运行
- Gateway 是否响应请求
- 是否有卡住的任务（超过 120 秒）
- 是否有连续超时（3 次以上）
- 是否有大量错误日志

**自动处理动作：**
- 进程不存在 → 自动启动 Gateway
- 无响应超过 30 秒 → 自动重启 Gateway
- 检测到卡住任务 → 自动发送停止命令
- 连续超时 → 自动重置会话
- 过期锁文件 → 自动清理

### 方案二：定时自动清理

定期清理可能导致卡顿的文件和数据。

**清理内容：**
- 删除超过 7 天的会话历史文件
- 压缩超过 100MB 的日志文件
- 删除超过 30 天的旧日志
- 清理临时锁文件
- 清理过大的会话文件（超过 10MB）

**执行频率：**
- 每天凌晨 3 点自动执行深度清理
- 或每小时执行一次轻量清理

**设置方法：**
1. 脚本已创建在：openclaw_auto_cleanup.sh
2. 添加到系统定时任务
3. 设置每天执行一次

**清理策略：**
- 会话文件超过 10MB → 备份后清理
- 日志文件超过 100MB → 压缩保存
- 会话文件超过 7 天 → 删除
- 日志文件超过 30 天 → 删除
- 锁文件超过 1 小时 → 删除

### 方案三：智能配置调整

根据使用情况自动调整配置，预防卡顿发生。

**调整策略：**
- 检测到频繁超时 → 自动降低超时时间
- 检测到 API 限流 → 自动禁用相关功能
- 检测到内存占用高 → 自动清理会话
- 检测到并发冲突 → 自动降低并发数

**实施方式：**
- 通过监控脚本检测问题
- 自动修改配置文件
- 发送重启命令使配置生效

**调整项：**
- 超时时间：从 600 秒降到 120 秒
- 并发数：从 4 降到 1
- 禁用慢速功能：web search、session-memory
- 清理会话历史：保持最近 100 条消息

### 方案四：健康检查机制

结合健康检查和自动恢复。

**检查频率：**
- 每 1-2 分钟检查一次
- 关键指标实时监控

**检查项：**
- Gateway 进程状态
- 端口监听状态
- 最近响应时间
- 活跃任务数量
- 资源使用情况

**恢复动作：**
- 进程不存在 → 启动服务
- 端口无响应 → 重启服务
- 检测到卡住 → 停止任务
- 资源耗尽 → 清理并重启

### 方案五：基于日志的智能检测

分析日志模式，识别卡顿特征并自动处理。

**检测模式：**
- 检测 agent.wait 超过阈值
- 检测连续的错误日志
- 检测超时模式
- 检测 API 限流模式
- 检测网络连接问题

**处理策略：**
- 单次超时 → 发送停止命令
- 连续超时 → 重置会话
- API 限流 → 禁用相关功能
- 严重问题 → 重启服务

**阈值设置：**
- agent.wait 超过 60 秒 → 警告
- agent.wait 超过 120 秒 → 自动停止
- 连续 3 次超时 → 重置会话
- Gateway 无响应 30 秒 → 重启

### 方案六：预防性维护

在问题发生前就进行维护，减少卡顿概率。

**维护内容：**
- 定期清理会话历史（保持最近 100 条）
- 定期压缩日志文件
- 定期检查配置合理性
- 定期验证 API 连接状态
- 定期检查磁盘空间

**执行时机：**
- 每天凌晨低峰期执行
- 或根据使用频率动态调整

**维护策略：**
- 会话历史超过 1000 条 → 清理旧记录
- 日志文件超过 100MB → 压缩
- 磁盘空间不足 → 清理旧文件
- 配置不合理 → 自动修正

### 方案七：通知和自动处理

检测到问题时，先通知用户，然后自动处理。

**通知方式：**
- macOS 系统通知
- 邮件通知
- WhatsApp 消息
- 日志记录

**处理流程：**
1. 检测到问题
2. 发送通知（可选）
3. 等待用户响应（可选，可设置超时）
4. 自动执行恢复操作
5. 记录处理结果

**通知内容：**
- 问题类型和严重程度
- 建议的处理方案
- 自动处理倒计时
- 处理结果反馈

## 推荐组合方案

### 轻量级自动化（适合个人使用）

**组合：**
- 自动监控脚本：每 5 分钟检查
- 定时清理脚本：每天一次
- 日志检测：实时监控

**效果：**
- 自动检测和处理卡顿
- 定期清理，保持系统健康
- 资源占用低

### 完整自动化（适合重度使用）

**组合：**
- 自动监控脚本：每 2 分钟检查
- 定时清理脚本：每小时轻量，每天深度
- 智能配置调整：实时
- 健康检查：每 1 分钟
- 日志检测：实时
- 预防性维护：每天

**效果：**
- 全方位自动化管理
- 快速响应和处理
- 预防性维护，减少问题

**操作：**

2. 添加到系统定时任务
3. 设置每天执行一次

根据实际使用情况，逐步添加其他自动化方案。

**操作：**

2. 针对性地添加自动化处理
3. 调整阈值和策略

## 监控指标建议

### 卡顿检测阈值

- agent.wait 超过 60 秒 → 警告记录
- agent.wait 超过 120 秒 → 自动停止任务
- 连续 3 次超时 → 重置会话
- Gateway 无响应超过 30 秒 → 重启服务

### 清理阈值

- 会话文件超过 10MB → 备份后清理
- 日志文件超过 100MB → 压缩保存
- 会话历史超过 1000 条 → 清理旧记录
- 日志文件超过 30 天 → 删除

### 性能阈值

- CPU 使用率超过 80% → 降低并发
- 内存使用超过 2GB → 清理会话
- API 错误率超过 10% → 禁用相关功能
- 磁盘使用超过 80% → 清理旧文件

## 注意事项

- 自动化脚本需要有适当的执行权限
- 避免过于频繁的检查，影响系统性能
- 保留足够的日志用于问题排查
- 设置合理的阈值，避免误判和误操作
- 重要操作前建议先备份数据
- 定期检查自动化脚本的运行状态
- 根据实际情况调整阈值和策略

## 日志和监控

### 监控日志位置

- 监控脚本日志：~/.openclaw/monitor.log
- 清理脚本日志：~/.openclaw/cleanup.log
- Gateway 日志：~/.openclaw/logs/gateway.log
- 错误日志：~/.openclaw/logs/gateway.err.log

### 查看日志

- 实时查看监控日志：tail -f ~/.openclaw/monitor.log
- 查看清理记录：tail -f ~/.openclaw/cleanup.log
- 查看最近操作：tail -50 ~/.openclaw/monitor.log

## 故障排查

### 如果自动化脚本不工作

1. 检查脚本权限：确保有执行权限
2. 检查定时任务：确认 crontab 配置正确
3. 检查日志：查看是否有错误信息
4. 手动测试：直接运行脚本测试

### 如果自动化处理过于激进

1. 调整检测阈值：增加时间限制
2. 增加确认步骤：处理前先通知
3. 降低执行频率：减少检查次数
4. 禁用某些自动操作：只保留必要的

### 如果自动化处理不够及时

1. 降低检测阈值：更早发现问题
2. 增加检查频率：更频繁地检查
3. 添加更多检测项：覆盖更多场景
4. 优化处理逻辑：更快地响应

## 总结

自动化方案可以大大减少手动干预的需要，但需要根据实际情况调整和优化。建议从基础监控开始，逐步添加其他功能，最终形成完整的自动化管理体系。
